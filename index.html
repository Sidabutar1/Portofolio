🍽️ Dokumentasi Sistem Saroha Catering

Sistem informasi terpadu untuk manajemen pemesanan paket katering dan penyewaan peralatan pesta (B2C & B2B). Sistem ini dirancang menggunakan arsitektur native PHP berkonsep MVC (Model-View-Controller) dengan PDO (PHP Data Objects) untuk interaksi database yang aman dan efisien.

🚀 Fitur Utama

1. Sistem Pemesanan (Customer / B2C)

  • Katalog Produk: Pelanggan dapat melihat paket katering dan peralatan yang tersedia.
  • Keranjang & Checkout Pintar: Hitung ongkos kirim otomatis berdasarkan berat (weight_kg), volume (volume_m3), jarak (distance_km), dan estimasi jumlah perjalanan (Trip) armada.
  • Manajemen Promo: Dukungan kode promo (minimum belanja, masa berlaku, kuota, persentase/fix).
  • Integrasi Pembayaran: Seamless dengan Midtrans (Snap API & Webhook) untuk e-Wallet, VA, QRIS, dll. Serta opsi Tunai.
  • Guest Order Tracking: Lacak status pesanan menggunakan Kode Pesanan & WhatsApp.

2. Manajemen Operasional (Staff Backend)

  • Dashboard Staff: Statistik ketersediaan aset & kalender jadwal (FullCalendar).
  • Quick Order Form (POS): Form cepat untuk pesanan tatap muka/telepon.
  • Manajemen Logistik: Pendaftaran real trips, plat kendaraan, dan konfirmasi delivery fee.
  • Notifikasi Terpadu: Email & WhatsApp otomatis ke Customer, Staff Logistik, dan Dapur.
  • Cek Ketersediaan: Validasi Overlap anti Double Booking.

3. Manajemen Eksekutif (Owner Backend)

  • Dashboard Keuangan: Statistik pendapatan & paket terlaris bulanan.
  • Laporan & Recycle Bin: Export `.CSV` dan fitur Soft Deletion untuk pesanan batal.
  • Manajemen Hak Akses: Akses berbasis Permissions (finance, fleet, kitchen).
  • Pengaturan Web: Pajak PPN, Midtrans Server Key, template WA, URL gambar default.

🛡️ Standar Keamanan Sistem

  1. Anti-SQL Injection: Menggunakan Prepared Statements (PDO).
  2. Anti-CSRF: Validasi token CSRF (di-generate per session) untuk aksi penting.
  3. Validasi Webhook Lapis Baja: Memvalidasi Signature Key (SHA512) dari notifikasi Midtrans.
  4. Proteksi Upload: Filter ekstensi (mime_content_type) & randomized renaming anti Web Shell.
  5. RBAC Middleware: Pemisahan endpoint logis antara Customer, Staff, dan Owner.

📂 Struktur Direktori Utama

saroha_catering/
├── api/                 # Endpoint REST/JSON API (Kalender, Webhook Midtrans)
├── config/              # Konfigurasi Database (Koneksi PDO) & Konstanta
├── controllers/         # Logika Utama Sistem
├── helpers/             # Fungsi bantu seperti UploadHelper
├── middleware/          # Pemeriksaan Role dan Hak Akses (AuthMiddleware)
├── models/              # Representasi tabel database & Model data
├── public/              # Aset publik, File Gambar, CSS, JS
├── services/            # Logika Bisnis Kompleks, Notifikasi WA, Email
└── views/               # Tampilan UI HTML/PHP
    ├── customer/        # UI Publik B2C
    ├── owner/           # UI Admin Dashboard Owner
    └── staff/           # UI Operasional Pesanan Staff

🔗 Modul Pihak Ketiga & Integrasi

  • Midtrans Payment Gateway: Snap API & Webhook server-to-server.
  • Layanan WhatsApp (NotificationService): Notifikasi tagihan & perintah tim.
  • Email Service (SMTP): Template dinamis list barang sewa dan paket.
  • FullCalendar (JS): Visualisasi pesanan dalam kalender yang bersih.

⚙️ Skema Database & Optimasi Indeks

Menggunakan indeks khusus (seperti idx_reservation_overlap) untuk mempercepat query hingga 50x lebih cepat. Relasi Entitas:

  • users
  • orders → order_items
  • equipment → equipment_reservations
  • promo_codes
  • logistics_tariffs

💡 Flow Pemesanan Online

  1. Customer memilih alat/paket dari katalog, mengisi detail checkout, ongkir & promo dihitung.
  2. Sistem men-generate No. Order, memvalidasi kalender, dan mengunci transaksi (pending).
  3. Customer membayar via Midtrans. Server memvalidasi Webhook dan status menjadi confirmed.
  4. Sistem mengirim WA resi dan notifikasi ke Staf. Staff mengalokasikan armada sebelum diantar.

🏥 Pengembangan Sistem Afya Hermina

Sistem informasi manajemen rumah sakit (Smart RCM System) dengan fokus pada optimalisasi siklus pendapatan Departemen Keuangan. Dirancang untuk mencegah kebocoran pendapatan (Revenue Leakage) dan percepatan arus kas melalui validasi cerdas dan otomatisasi.

🚀 Fitur Unggulan untuk Keuangan & RCM

1. Sistem Smart Gatekeeping

Mencegah klaim asuransi ditolak (Return to Provider) karena berkas tidak lengkap.

  • File Lock Mechanism: Otomatis mengunci gerbang Kasir jika berkas Lab/Radiologi belum diunggah oleh Unit Medis.
  • Dampak: Memastikan 100% Clean Claim sebelum data masuk ke Departemen Keuangan.

2. Billing Lock & Auto-Split Tagihan

  • Real-time Limit Detection: Memblokir pencetakan kwitansi jika tagihan medis melampaui plafon asuransi.
  • Auto-Split Logic: Memisahkan otomatis tagihan asuransi dan selisih (excess) yang ditagihkan tunai ke pasien.

3. Dashboard Pemantauan Piutang (Aging AR)

  • Kategorisasi Umur: Hijau (0-30 Hari), Kuning (31-60 Hari), Merah (>60 Hari Kritis).
  • Visualisasi Data: Integrasi ChartJS distribusi status klaim & Top 5 Asuransi penunggak terbesar.

4. Smart Import CSV & Automasi Data

  • Fuzzy Logic Matching: Mencocokkan nama asuransi dari file CSV Excel meski terdapat typo.

5. Otomatisasi Reminder Penagihan

  • Template Dinamis: Pesan WA/Email dinamis dengan variabel {full_name}, {nominal}.
  • Penjadwalan follow-up berkala setiap 14 hari pasca-klaim.

💡 Alur Keuangan yang Disempurnakan (Workflow)

  1. Pendaftaran (Unit A): Pengecekan limit asuransi divalidasi sejak awal masuk.
  2. Medis (Unit B): Input tindakan medis otomatis terkonversi rupiah tanpa intervensi kasir.
  3. Kasir (Unit C): Melewati Gatekeeping (cek berkas & limit plafon). Menghasilkan tagihan Clean Claim.
  4. Keuangan (Unit E): Menerima data piutang bersih, mengeksekusi tagihan asuransi, memantau umur piutang, dan menetapkan pelunasan.

🛠️ Arsitektur Teknis & Struktur Database

  • Native PHP OOP
  • Vanilla JS
  • Tailwind CSS
  • MariaDB
  • Chart.js
  • Service Workers (PWA)

Desain Relasional Model:

  • transactions & transaction_details (Pencatatan siklus medis dan nominal subtotal)
  • transaction_attachments (Path berkas medis sebagai syarat buka File Lock)
  • claims (Sentral RCM pelacak status pembayaran piutang bersih)
  • claim_reminders & message_templates (Automasi penagihan)

🎯 Ringkasan Nilai Bisnis

Sistem Afya Hermina bertindak sebagai "Penjaga Gerbang Keuangan". Secara cerdas mendikte kedisiplinan pergerakan administrasi, sehingga Departemen Keuangan hanya menerima data yang sudah terkurasi, valid, dan memiliki peluang sukses pencairan klaim asuransi mendekati 100%.

  • **Tata Letak:** Mengoptimalkan ruang untuk informasi kontak esensial seperti nama, jabatan, nomor telepon, email, dan alamat website/media sosial.

  • **Tata Letak:** Desain yang ringkas dan fokus pada informasi kunci, memastikan kemudahan aksesibilitas.
